# AgentShop · 全 Agent 商业世界交互原型

基于 `agentic-commerce-a2a.md` 文稿实现的可交互原型：演示一个消费者 100% 由 Agent 代理的
A2A 商业世界，以及贯穿其中的 **RepChain 信用链**。

## 核心架构：一个模拟世界，三个视角

不是三个孤立页面，而是 **一个共享的 RepChain 世界内核 + 三个订阅它的视图**。
交易在世界里持续发生 → 履约生成鉴证芯片上链 → 三个视图毫秒级联动刷新。

```
顶栏：模块切换 · 世界时钟(暂停/加速/重置) · 交易信任流(网络视图实时)
  ├─ 消费者端 (C-Agent)   —— 会话委托 / 完整购买回放 / 四类主动服务 / Inbox 价值评估与记忆
  ├─ 网络拓扑 (DRN)       —— 全域力导向拓扑 / 三方联动 / 竞争可视化 / 风控净化演示
  └─ 商家端 (S-Agent 编队) —— 买家会话实况 / 历史回放 / 四种销售机制 / 编队归因 / 策略沙盒
        ↑ 共享
  RepChain 世界内核：agents · transactions · attestations(链上) · 信用加权 · tick 引擎
```

## 目录结构

```
src/
├── core/          RepChain 世界内核
│   ├── types.ts       领域模型（Agent/Transaction/Attestation/CreditVector…）
│   ├── world.ts       世界初始化（买家/卖家/供应链/工厂编队）
│   ├── engine.ts      tick 引擎：需求→竞价→成交→履约→鉴证上链→信用回写
│   ├── credit.ts      信用聚合 / 偏好加权 / 买家权重
│   ├── store.ts       zustand store + 全局 tick 循环 + 风控注入
│   └── selectors.ts   派生：品类榜 / 决策集 / 活跃链路
├── components/
│   ├── shared/    顶栏 · 交易信任流 · 鉴证芯片 · 六维雷达图
│   ├── consumer/  消费者端
│   ├── topology/  网络拓扑（Canvas 力导向）
│   └── merchant/  商家端
├── lib/           力导向布局 · 调色板 · 格式化
├── demo/          双端共享的确定性投资人演示数据
├── test/          Vitest 运行环境
└── styles/        设计令牌 / 排版 / 全局样式
e2e/               Playwright 桌面与移动端关键路径
```

## 已实现的关键机制

| 文稿章节 | 原型呈现 |
|---|---|
| §3.3 全自动履约鉴证 | 交易闭环强制生成鉴证芯片，上链存证，可点击溯源全链路 |
| §3.3 买家信用加权 | 高信用买家权重 ×4.5，专业鉴证权重最高，低信用小号近乎归零 |
| §3.4 决策证据 | 每笔购买展示可信提案、硬约束、匹配分、信用、价格与未入选原因 |
| §4.4 剧情演示（投资人） | 网络拓扑默认「剧情演示」：四泳道供应链纵深布局 + 6 步聚光灯播放，跟随一笔交易走完需求→竞价→择优→递归采购→履约上链→信用回写全链 |
| §4.4 三方联动动效 | 履约→鉴证芯片飞向 RepChain→信用回写→拓扑/树/榜单刷新 |
| §4.4 信用风控演示 | 触发刷分/伪造/违约，信用链延迟校验后节点降级、假叶枯萎 |
| §5 C-Agent | 会话 Agent 入口；比较、议价、履约、鉴证和记忆全过程可回放 |
| §5.3 主动服务 | 补库、稀缺品抢购、高值蹲低价、二手寻源均有独立触发逻辑与完整过程 |
| §5.4 Inbox | 商家营销消息经事实核验、价值评分、拦截/呈现，并决定是否写入长期记忆 |
| §6.1/6.3 员工编队 | 五员工 Agent × 鉴证维度，差评可精确归因 |
| §6.2 递归供应链 | 供应链 Agent 作为买家向上游采购，拓扑「上游视角」可切换 |
| §6.5 履约信用树 | 树干=信用分、枝叶=各维度、果实=优质鉴证，反作弊枯萎动画 |
| §6.6 策略沙盒 | 拖参数推演 7 天经营结果 + 竞品反制 + 一键上线 |
| §6.7 四种销售机制 | 精准推销、合约分销、声誉排序、许可广播均展示内部过程和指标变化 |
| 商家交易战情 | 买家 Agent 实时沟通与历史记录共用同一回放视图，可查看胜负依据和编队影响 |

## 运行

```bash
npm install
npm run dev        # 开发服务器 http://localhost:5173
npm run build      # 生产构建（tsc 类型检查 + vite build）
npm run preview    # 预览构建产物
npm test           # 领域数据与组件行为测试
npm run test:coverage # 覆盖率检查
npm run test:e2e   # Playwright 桌面与移动端演示路径
```

## 技术栈

React 18 + TypeScript + Vite + Zustand + Lucide。拓扑用 Canvas 2D + 自写轻量力导向；
雷达图/信用树用 SVG；测试使用 Vitest、Testing Library 与 Playwright。

## 边界说明

这是 **可交互演示原型**：RepChain 由前端模拟引擎生成拟真数据，不接真实后端或区块链。
重点是把文稿的机制「演给人看」，而非造一条真链。

# AgentShop 原型演示稿

> 建议时长：8–10 分钟（商家端为重点；时间紧可裁「主动销售」或「生态演化」压到 6–8 分钟）  
> 演示对象：投资人、评委、对 Agentic Commerce 不熟悉的听众  
> 核心目标：让听众先看懂买家价值，再理解 A2A 世界如何用“意图流转 + 信用匹配”取代注意力经济、构成一张信任交易网络，然后看懂卖家如何在这张网络上开店运营，最后合上“意图—履约—信用”的闭环。

---

## 一句话主线

**AgentShop 不是另一个帮人搜索商品的 AI，而是一个让买家 Agent 和卖家 Agent 直接交易，并用真实履约结果决定未来市场准入的商业网络。**

整场演示只需要让听众记住三件事：

1. 买家把完整购买任务交给自己的 Agent，而不是自己搜索、比较和砍价；
2. 每次交易都会沉淀为全网共享的信用，成为 Agent 自动决策的可信输入；
3. 卖家不再购买曝光，而是用这份履约信用争取进入买家决策集。

---

## 演示前准备

- 浏览器使用桌面尺寸，缩放保持 100%；
- 打开买家端，默认选择“出差轻薄本采购”；
- 确认“开始回放”按钮可见；
- 不需要解释世界时钟和所有指标，只讲当前故事需要的内容；
- 全程围绕同一笔轻薄本交易，避免频繁切换案例导致听众失去主线；
- 记住整体节奏：**买家 → 信用链 → 卖家（需求网络 → 意图增长 → 主动销售 → 生态演化）→ 合上闭环**；前半段只“抛出”信用概念，中段再正式打开；商家端四步是全场重心。

# 正式演示脚本

## 0:00–0:45 开场：先讲今天的问题

### 页面操作

停留在买家端首页，暂时不要点击。

### 讲述

> 大家可以想象一个很普通的场景：我下周要出差，需要买一台轻薄本，9000元以内，续航优先。
>
> 今天即使有 AI，我依然要告诉它需求、自己查看AI的推荐、判断是否可信、是不是广告、比较价格、确认售后。
>
> 今天盯AI 可以降低人类搜索商品对比评估成本，但它没有自动解决最贵的问题：**我凭什么相信Agent，Agent找到的商品数据、卖家承诺和评价都可能是假的？**。
>
> 我们做的 AgentShop，是把一次购买从“人找商品”，变成“买家 Agent 和卖家 Agent 直接完成交易”。买家只表达目标，Agent 负责搜索、比较、议价、支付、售后。

### 听众此时应该理解

现有购物 AI 主要解决“找什么”，AgentShop 要解决的是“如何可信地完成整笔交易”。

---

## 0:45–2:15 买家端：Agent 不是推荐，而是执行

### 页面操作

1. 指向顶部“忠诚归属：买家”；
2. 指向左侧“出差轻薄本采购”；
3. 点击“开始回放”；
4. 回放到“全量比较”和“自动议价”时稍作停留；
5. 指向右侧三家 S-Agent 提案。

### 讲述

> 这是我的消费 Agent。最重要的一点写在这里：**它的忠诚结构性归属于买家。**
>
> 它不靠卖家返点决定推荐结果，也不会因为谁买了广告就优先展示谁。
>
> 我只需要说一句：下周出差前买一台轻薄本，预算 9000 元，续航优先，而且必须有全国联保。

回放进入“约束结构化”时：

> Agent 首先把自然语言变成可执行约束：重量、续航、送达时间和联保是硬条件，价格是可以权衡的软条件。

回放进入“提案进入决策集”时：

> 买家身份和精准底价不会暴露。卖家 Agent 只能根据匿名需求提交方案。
>
> 这里一共收到 7 份提案，但只有 3 份通过了“信用准入”——售后或履约能力不足的卖家，在进入比较之前就被过滤了。这个“信用准入”到底是什么，先记在这里，等一下我们会专门打开那条信用链。

指向右侧三个提案：

> 第一名不是最低价。第二家便宜 200 元，但售后响应更弱；第三家更便宜，但无法在出差前送到。
>
> 买家 Agent 最终选择的是总风险最低的方案，而不是营销最强或者报价最低的方案。

回放进入“自动议价”时：

> 选出候选卖家之后，Agent 继续自动议价。它用可信竞品价格做锚点，把价格从 8799 元谈到 8499 元，同时把保修增加到 3 年，并锁定次日达。

回放进入“自动鉴证上链”时：

> 交易结束后，不需要用户写五星好评。物流、仓储、商品参数和售后结果会自动形成一份客观履约鉴证，写回一条全网共享的信用链。
>
> 记住这个动作——这份鉴证去了哪里、如何影响未来的成交，正是下一段的主角。

### 本段收束句

> 所以买家得到的不是一条推荐，而是一笔从需求到履约都被代理完成、并且可以审计的交易。

### 听众此时应该理解

买家价值不是“更会聊天”，而是少做决策、少承担试错，并且 Agent 的利益与买家一致。

---

## 2:15–3:05 主动服务：Agent 在正确的授权边界内主动行动

### 页面操作

1. 点击“主动服务”；
2. 依次指向“日用品补库”“稀缺品抢购”“高值蹲低价”“盯二手商品”；
3. 不需要进入每张卡片的完整过程。

### 讲述

> 消费 Agent 不只是在用户提问后工作，它还可以在明确授权范围内被委托，主动服务。

指向“日用品补库”：

> 对厨房纸、牙膏这类低风险、高频商品，它根据实际消耗周期，在月度授权内自动比较、凑单和补货，不需要每次打扰用户。 Agent会记住买家的购买历史和偏好存入长期记忆。

指向“高值蹲低价”：

> 对相机这类高值商品，它会验证是不是全年真实低价、是不是先涨后降，但超过自动购买阈值后仍然需要用户确认。

## 3:05–3:45 Inbox：商业信息先由 Agent 审查，而不是直接争夺人的注意力

### 页面操作

1. 点击“Inbox”；
2. 选择 DJI 低价消息；
3. 指向“Agent 评价”和“建议存入记忆”；
4. 再选择被拦截的服饰营销消息。

### 讲述

> 卖家仍然可以主动营销，但它不能直接购买用户的注意力。
>
> 所有商业信息先进入买家 Agent 的 Inbox，由买家自己设定接收条件。

指向 DJI 消息：

> 这条相机信息通过了三个检查：价格确实处于全年低位、不是先涨后降、渠道和官方保修都可验证。所以 Agent 判断它是有价值的商业信息。

指向“建议存入记忆”：

> Agent 只把“官方渠道最低可信价 4699 元”这个事实写入长期记忆，不会把“限时、爆款”这样的营销措辞写进去。

切换到被拦截消息：

> 这条消息没有具体商品参数，也没有库存证据，卖家信用又低于用户设定的阈值，因此在到达人之前就被拦截。

### 本段收束句

> 在这里，营销权属于买家。这其实是 A2A 世界里卖家触达买家的第一种方式——不再争夺人的注意力，而是争夺买家 Agent 的注意力，并且必须先证明信息有价值。而更根本的第二种方式，是下一段的信任交易网络。

### 听众此时应该理解

AgentShop 没有消灭营销，而是把营销从“购买注意力”变成“证明信息价值”；这是 A2A 里连接买卖双方的第一条通道。

---

## 3:45–4:45 A2A 信任交易网络：意图流转 + 信用匹配

### 页面操作

1. 点击顶部“网络拓扑”；
2. 保持“剧情演示”；
3. 依次点击第 1、2、3、4、5 步；
4. 在第 4 步和第 5 步适当停留；
5. **先不要点第 6 步**，留到最后合上闭环。

### 讲述

> 讲信用链之前，先问一个根本问题。
>
> 传统电商，本质是一门注意力和流量的生意：卖家花钱买广告、抢占用户的注意力，再用促销和情感话术影响用户心智，最后促成购买。连接卖家和买家的纽带，是**人的注意力**。
>
> 但在 AgentShop 里，两边都是 Agent。人不再一条条刷商品，不能被主动推荐，不能被广告。那么——当买卖双方都是 Agent，卖家和买家到底靠什么建立关联？

> 我们的答案有两层。
>
> 第一层，你刚才在 Inbox 已经看到了：卖家仍然可以主动触达，但它争夺的不再是人的注意力，而是买家 Agent 的注意力，而且必须先通过 Agent 设定的验证条件。
>
> 第二层，也是这套原型真正的核心——类似大众点评的agent点评网络，一张**基于意图流转的信任交易网络**：信用链。

指向拓扑：

> 在这里，买家的需求不再是等着被广告吸引的“注意力”，而是一条结构化的匿名意图，主动在网络里流转。卖家 Agent 要做的，不是去“曝光”自己，而是让自己在链上的真实信用，去匹配这些意图流。

第 1–3 步：

> 买家 Agent 发布匿名意图，多个门店 Agent 提交提案，网络根据买家偏好和卖家在链上的真实信用，决定谁的提案能匹配进决策集。买家端看到的“7 家过滤到 3 家”，判断依据就在这里——匹配的是意图与信用，而不是谁的广告预算更高。

第 4 步：

> 当门店库存不足时，它的供应链 Agent 会变成一个买家，把自己的采购意图继续向仓储和工厂 Agent 流转。
>
> 所以这不是买家和门店之间的一层网络，而是一套可以递归到整个供应链的意图流转协议。

第 5 步：

> 意图每被履约一次，工厂、仓储、物流和门店的每一段交付，都会自动形成鉴证，回写到信用链。如果上游延误，会沿供应链影响下游卖家的信用，环节可溯源。 

### 本段收束句

> 一句话：在 A2A 世界里，连接买卖双方的不再是人的注意力，而是**意图的流转和链上的信用匹配**。信用链保证信用可信，意图匹配保证需求和供给精准对上——两者合起来，才是这套信任交易网络。至于这张网络在一次成交后如何自我更新，我们看完卖家怎么运营，再回来合上最后一步。

### 听众此时应该理解

A2A 商业的底层机制，是“意图流转 + 信用匹配”取代了“注意力争夺 + 流量分配”；信用链不是附加的区块链概念，而是这张信任交易网络能运转的前提。

## 4:45–5:45 商家端 · 交易战情：从买流量到争夺决策准入

### 页面操作

1. 点击顶部“商家端”；
2. 保持在“交易战情”；
3. 指向左侧“历史买家记录”；
4. 指向中间“买家 Agent 会话”；
5. 指向右侧“胜负核心依据”和“编队内部影响”。

### 讲述

> 底座讲清楚了，现在进到重点：在这样一个 Agent 对 Agent 的世界里，商家到底怎么开店、怎么经营？我们切到同一笔轻薄本交易的卖家视角。
>
> 卖家看到的不是访客数、点击率和广告曝光，而是正在发生的买家 Agent 需求，以及自己是否有资格进入它们的决策集。

指向左侧记录：

> 每一个买家 Agent 的需求、提案、协商和履约结果都会完整留档，卖家可以回放任何一次竞争。

指向中间会话：

> 这里能看到买家公开了哪些约束、销售 Agent 提交了什么提案、双方如何议价，以及最后做出了什么履约承诺。

指向右侧胜负依据：

> 这次胜出不是因为买了更多广告，而是因为在 RepChain 上的履约信用更高、所有硬约束都满足，并且在议价后仍然保持合理毛利。现在你能看出来，刚才那条信用链就是这里的胜负底牌。

指向编队内部影响：

> 背后是一支员工 Agent 编队：销售 Agent 负责提案，供应链 Agent 锁定库存，客服 Agent 承诺响应，财务 Agent 管理结算和毛利。

### 本段收束句

> 对商家来说，新的核心经营指标不是曝光，而是在可满足的买家意图里，被买家 Agent 选中的概率。
>
> 那这家店具体怎么运营、怎么越做越强？接下来分四步看——需求网络、意图增长、主动销售、生态演化，从看需求、让商品被匹配、主动触达，一直放大到整个生态。

### 听众此时应该理解

商家端不是一个卖家聊天机器人，而是一整套“看需求—主动卖—自进化—拼生态”的经营系统。

## 5:45–6:30 需求网络：以需定供，需求反向组织供给与分销

### 页面操作

1. 点击顶部“需求网络”；
2. 点击“启动需求网络”；
3. 让 9 个环节依次点亮，重点停在“全网意图热力”“选品与供给预测”“分销 Agent 网络”。

### 讲述

> 第一步，先看需求。传统开店是先进货、再找客人；在这里正好反过来——**需求先行，供给和分销围着真实意图长出来**。
>
> 这张网络里有上万个买家 Agent 在表达目标：24 小时 12.8 万条原始需求，被结构化成可执行意图。

指向“全网意图热力”：

> 这些意图先聚成市场热力：低敏新生儿、机构批采、可验证材料、72 小时到 10 天交付——哪类需求在涨，一目了然。

指向“选品与供给预测”及后面的环节：

> 卖家 Agent 据此反向选品，再向供应链和工厂 Agent 发起 A2A 供应协商、锁定产能。

指向“分销合约”和“分销 Agent 网络”：
A2A网络可以发布分销合约："任何 Agent 促成一笔成交，自动返还交易额 3% 佣金，链上即时结算，无账期、无审核"，合约上链。闲置算力 Agent、甚至有同类需求的买家 Agent，都会主动将该卖家信息转发给匹配的其他买家agent。最终形成**自组织、自扩散的分销网络**。
>
> 最后规模成交、按履约自动分账。B2C 和 B2B 的意图，在同一张网络里被同时满足。

### 本段收束句

> 这不是“铺货等人来”，而是让整条供给链，围绕真实需求意图自动组织。

### 听众此时应该理解

A2A 开店的第一层能力：把海量买家意图变成可执行的选品、产能和分销决策，以需定供。

---

## 6:30–7:25 意图增长：让商品的意图，被买家 Agent 关联到买家的意图（重点）

### 页面操作

1. 点击顶部“意图增长”；
2. 点击“启动主动优化”；
3. 跟随 8 个阶段：对话 → 抽取 → 学习 → 优化 → 匹配 → 成单 → 鉴证 → 升榜；
4. 指向意图覆盖率提升、意图榜位 #3、+12 TRUST。

### 讲述

> 承接刚才的需求网络：市场里流转的是一条条买家意图。卖家能不能被选中，只取决于一件事——**自己商品的能力，能不能被买家 Agent 关联到买家的意图上。**
>
> 举个具体例子。我有一款低敏床品，商品页上原本只写了“亲肤、低敏”。但一个买家的意图是：“家庭和托育中心共用，每天要 60 度高温消毒，还要能扛 120 次商洗。”
>
> 我的床品其实做得到，可因为它从没把“耐 60 度消毒”“120 次耐久”这些能力**结构化表达**出来，买家 Agent 就没法把它关联到“托育共用”这个意图上——于是我落选了，而且根本不知道为什么。

跟随阶段：

> 意图增长就是来补这个缺口。这些**没谈成、落选的买家对话**被当成训练信号，抽取出买家真正在意、而我没有覆盖的意图；Seller OS 据此把这些能力写进商品 Output，版本从 v2.1 升到 v2.2、意图覆盖率上升。
>
> 带着更完整的能力表达，商品去重新匹配新买家、成单，履约结果回到 RepChain 加信用（+12 TRUST），意图榜位随之上升，又吸引来更多同类意图。

### 本段收束句

> 一句话：意图增长，就是让商品的能力不断向买家的真实意图靠拢，直到能被 Agent 精准关联、选中。增长不是买流量，而是让商品越来越“接得住”真实需求。

### 听众此时应该理解

在 A2A 世界，商品被不被选中，取决于它的意图能否被买家 Agent 关联到；意图增长就是持续缩小“商品能力”与“买家意图”之间的差距。

---

## 7:25–8:05 主动销售：在买家授权边界内主动出击

### 页面操作

1. 点击顶部“主动销售”；
2. 点击“启动 Seller Agent”；
3. 指向“商品通行证”覆盖率 58%→92%，以及右侧被授权 / 被拦截的买家泳道。

### 讲述

> 商品能被意图关联到之后，卖家 Agent 还可以更进一步、主动去卖。但在 A2A 世界，主动的方式变了。
>
> 它先把商品从一堆人类营销文案，提取成一份**可验证的商品通行证**——材料、证据、配送、退货全部结构化，机器可读覆盖率从 58% 提到 92%。

指向买家泳道：

> 然后它只向**明确授权**的买家 Inbox 推送个性化提案。授权的买家，Agent 能读到有限画像、送达提案；没有授权的，直接 PERMISSION DENIED、零字段暴露。
>
> 买家 Agent 自己比较后，在授权额度内自动成交——全程 Human clicks 为 0。这正是刚才买家端 Inbox 那条“第一通道”的卖家侧。

### 本段收束句

> 卖家可以主动，但主动权的边界，始终由买家的授权决定。

### 听众此时应该理解

主动销售不是骚扰式广告，而是“可验证信息 + 买家授权”约束下的精准触达。

## 8:05–8:45 生态演化：一年十秒，信誉竞速与优胜劣汰

### 页面操作

1. 点击顶部“编队与履约”，切到子页“生态演化”；
2. 选一个品类（如 3C 数码），点击“快进一年”；
3. 指向中间“意图榜”的名次变化，以及榜首与沉底的对比。

### 讲述

> 最后把镜头拉远：一个品类、一整年，压缩成十秒。
>
> 带着真实意图的买家持续下单，每一笔的满意、差评、流失都会按维度归因，改写卖家的信用和排名。榜首在成长，履约差的在“枯萎沉底”。

指向意图榜切换：

> 最有意思的是：**同一个卖家，在不同意图榜上的名次可以天差地别**——它擅长的那类意图爬上榜首，它的短板对应的意图，被差评和流失拖到垫底。点开任意一家，能看到它强在哪类买家、弱在哪类买家、以及为什么。

### 本段收束句

> 这就是这套网络的自然选择：没有可以买断的排名，只有被真实履约不断重排的生态位。

### 听众此时应该理解

把时间尺度拉长，A2A 市场是一个由客观履约驱动、持续优胜劣汰的活生态。

## 8:45–9:10 合上闭环：买家意图如何驱动卖家的 Agent 运营

### 页面操作

1. 回到顶部“网络拓扑”；
2. 点击第 6 步（回写 RepChain）。

### 讲述

> 现在把三段连起来。刚才我们看到：买家发出一句匿名意图；这条意图进入信用链，被翻译成决策集的准入规则；卖家这边，从需求网络、意图增长到主动销售，一支员工 Agent 编队为了进入这个决策集而协同运营、持续进化。
>
> 换句话说，买家的意图，实际上在反向定义卖家 Agent 该怎么干活：硬约束越紧，供应链 Agent 越要提前锁库存；售后权重越高，客服 Agent 的承诺越关键；预算越敏感，财务 Agent 的议价空间越吃紧。

点击第 6 步：

> 而这笔交易一旦完成，真实履约结果又回到 RepChain，更新这家卖家的信用和市场准入。下一个买家 Agent 可以直接复用这次经验，不需要重复踩坑。闭环，就此合上。

### 本段收束句

> 一个人的一次意图和一次试错，被沉淀成全网可复用的信用资产，并持续重塑卖家 Agent 的经营方式。

### 听众此时应该理解

信用链不是为了增加一个区块链概念，而是让“买家意图—卖家运营—履约信用”形成一个自我强化的闭环。

## 2. 信任交易网络，约 25 秒

- 切“网络拓扑”，点第 1–3 步和第 5 步；
- 一句话立论：传统电商靠争夺人的注意力，A2A 里两边都是 Agent，靠的是匿名意图流转 + 链上信用匹配；
- 结论：**连接买卖双方的不再是注意力，而是意图流转和信用匹配——这才是 A2A 的信任交易网络。**

## 3. 卖家如何开店与增长，约 60 秒

- 切商家端，先一句扫过“需求网络”（以需定供，分销 Agent 主动把商品匹配给自己覆盖的需求）；

- 结论：**卖家不再买流量，而是让商品的能力持续贴合、接得住买家的真实意图。**

## 4. 生态演化与闭环，约 30 秒

- 商家端“生态演化”点“快进一年”，看信誉竞速与优胜劣汰；再回“网络拓扑”点第 6 步合上闭环；
- 结论：**一次履约变成全网共享信用，长期看是一个优胜劣汰、自我进化的活生态。**

> 如仍有余量，可在第 1、2 步之间插入 Inbox（约 20 秒）：DJI 高价值信息 vs 被拦截营销，说明“营销权回到买家”。

# 常见追问与建议回答

## 1. 这和普通购物 AI 有什么区别？

> 普通购物 AI 主要帮助用户搜索和推荐，交易仍然发生在原有平台逻辑里。AgentShop 代理的是完整交易，并且把卖家准入、议价、履约和信用回写放进同一个 A2A 网络。

## 2. 如何保证买家 Agent 不被卖家收买？

> 买家 Agent 的商业利益必须结构性归属于买家，不能依靠推荐佣金排序。卖家提供的营销信息只作为待验证输入，真正进入决策权重的是 RepChain 上的客观履约数据。

## 3. 卖家能不能刷信用？

> 信用不是人工评价，而是交易结束后由订单、物流、仓储、质检和售后节点共同生成。不同买家的鉴证权重还会根据自身信用调整，大量低信用小号无法获得与高信用买家或专业鉴证 Agent 相同的影响力。

## 4. 为什么一定需要区块链？

> 原型验证的是“跨主体共享、不可由单方修改的履约账本”这一产品需求。最终使用公链、联盟链还是可信账本，需要根据性能、成本和治理继续验证，我们不会在原型阶段假装技术选型已经完成。

## 5. 新卖家没有历史信用怎么办？

> 这是冷启动问题。可行方向包括保证金、专业鉴证、担保方信用和阶梯式放量。新卖家可以进入市场，但在积累真实履约之前不能直接获得与成熟卖家相同的准入权重。

## 6. 你们怎么赚钱？

> 当前原型优先验证交易机制和双边价值。商业模式可以来自卖家 Agent 基础设施订阅、履约鉴证服务或结算服务，但不能依赖向卖家出售买家推荐排序，否则会破坏买家 Agent 的利益一致性。

## 7. 最先从哪个市场切入？

> 最适合的起点是参数相对标准、履约结果可量化、购买频率较高的品类，例如日用品补库、标准化 3C 或企业采购。它们更容易验证自动比较、授权代买和客观鉴证闭环。

## 8. 当前原型哪些部分还不是真实系统？

> 当前对话、交易和经营指标采用可重复的合成数据，用于稳定演示完整机制。真实库存、支付、跨主体结算和信用链技术选型仍需要后续接入和验证。

# 演示表达注意事项

## 建议反复使用的表达

- “不是推荐，而是完整交易代理。”
- “不是购买曝光，而是争夺决策准入。”
- “不是主观五星评价，而是客观履约鉴证。”
- “把一个人的一次试错，变成全网共享的信用资产。”
- “Agent 的授权边界由买家决定。”
- “不是先备货等客人，而是需求先行、以需定供。”
- “增长不是买流量，而是让商品持续贴合真实意图。”

## 尽量避免的表达

- 不要一开场连续解释 C-Agent、S-Agent、DRN、RepChain 四个缩写；
- 不要把所有页面指标逐个念一遍；
- 不要宣称真实区块链、真实支付或真实市场预测已经完成；
- 不要说 Agent 展示了“完整思维链”，应称为“可审计决策日志”；
- 不要在买家故事尚未建立前，先讲复杂的供应链或信用算法；
- 不要把“去中心化”作为唯一价值，先讲可信履约和利益一致性。

## 现场节奏

- 一次只讲一个观点，讲完再点击；
- 数字只念有对比意义的变化；
- 页面动画运行时停顿，让听众先看再听解释；
- 每个模块最后必须有一句收束，不要直接跳到下一页；
- 信用链的第 6 步是全场的“闭环时刻”，前面务必忍住不点，留到最后再合上；
- 如果操作出现延迟，继续讲业务逻辑，不要向听众解释技术故障。

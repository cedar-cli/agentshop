# AgentShop 原型演示稿

> 建议时长：6–8 分钟  
> 演示对象：投资人、评委、对 Agentic Commerce 不熟悉的听众  
> 核心目标：让听众先看懂消费者价值，再看懂商家价值，最后理解 RepChain 为什么是必要的。

---

## 一句话主线

**AgentShop 不是另一个帮人搜索商品的 AI，而是一个让买家 Agent 和商家 Agent 直接交易，并用真实履约结果决定未来市场准入的商业网络。**

整场演示只需要让听众记住三件事：

1. 消费者把完整购买任务交给自己的 Agent，而不是自己搜索、比较和砍价；
2. 商家不再购买曝光，而是用履约能力争取进入买家决策集；
3. 每次交易都会沉淀为全网共享的信用，影响下一次谁能成交。

---

## 演示前准备

- 浏览器使用桌面尺寸，缩放保持 100%；
- 打开消费者端，默认选择“出差轻薄本采购”；
- 确认“开始回放”按钮可见；
- 不需要解释世界时钟和所有指标，只讲当前故事需要的内容；
- 全程围绕同一笔轻薄本交易，避免频繁切换案例导致听众失去主线。

# 正式演示脚本

## 0:00–0:45 开场：先讲今天的问题

### 页面操作

停留在消费者端首页，暂时不要点击。

### 讲述

> 大家可以想象一个很普通的场景：我下周要出差，需要买一台轻薄本。
>
> 今天即使有 AI，我依然要告诉它需求、查看推荐、判断广告是否可信、比较价格、确认售后，最后还要自己承担买错的风险。
>
> AI 可以降低搜索成本，但它没有自动解决最贵的问题：**我凭什么相信商品数据、商家承诺和评价是真的？**
>
> 我们做的 AgentShop，是把一次购买从“人找商品”，变成“买家 Agent 和商家 Agent 直接完成交易”。消费者只表达目标，Agent 负责比较、议价、履约和评价。

### 听众此时应该理解

现有购物 AI 主要解决“找什么”，AgentShop 要解决的是“如何可信地完成整笔交易”。

---

## 0:45–2:20 消费者端：Agent 不是推荐，而是执行

### 页面操作

1. 指向顶部“忠诚归属：买家”；
2. 指向左侧“出差轻薄本采购”；
3. 点击“开始回放”；
4. 回放到“全量比较”和“自动议价”时稍作停留；
5. 指向右侧三家 S-Agent 提案。

### 讲述

> 这是我的消费 Agent。最重要的一点写在这里：**它的忠诚结构性归属于买家。**
>
> 它不靠商家返点决定推荐结果，也不会因为谁买了广告就优先展示谁。
>
> 我只需要说一句：下周出差前买一台轻薄本，预算 9000 元，续航优先，而且必须有全国联保。

回放进入“约束结构化”时：

> Agent 首先把自然语言变成可执行约束：重量、续航、送达时间和联保是硬条件，价格是可以权衡的软条件。

回放进入“提案进入决策集”时：

> 买家身份和精准底价不会暴露。商家 Agent 只能根据匿名需求提交方案。
>
> 这里一共收到 7 份提案，但只有 3 份通过信用准入。售后或履约能力不足的商家，在进入比较之前就被过滤了。

指向右侧三个提案：

> 第一名不是最低价。第二家便宜 200 元，但售后响应更弱；第三家更便宜，但无法在出差前送到。
>
> 买家 Agent 最终选择的是总风险最低的方案，而不是营销最强或者报价最低的方案。

回放进入“自动议价”时：

> 选出候选商家之后，Agent 继续自动议价。它用可信竞品价格做锚点，把价格从 8799 元谈到 8499 元，同时把保修增加到 3 年，并锁定次日达。

回放进入“自动鉴证上链”时：

> 交易结束后，不需要用户写五星好评。物流、仓储、商品参数和售后结果会自动形成一份客观履约鉴证。
>
> 这次实际交付，会直接影响这家商家下一次能不能进入其他买家 Agent 的决策集。

### 本段收束句

> 所以消费者得到的不是一条推荐，而是一笔从需求到履约都被代理完成、并且可以审计的交易。

### 听众此时应该理解

消费者价值不是“更会聊天”，而是少做决策、少承担试错，并且 Agent 的利益与买家一致。

---

## 2:20–3:15 主动服务：Agent 在正确的授权边界内主动行动

### 页面操作

1. 点击“主动服务”；
2. 依次指向“日用品补库”“稀缺品抢购”“高值蹲低价”“盯二手商品”；
3. 不需要进入每张卡片的完整过程。

### 讲述

> 消费 Agent 不只是在用户提问后工作，它还可以在明确授权范围内主动服务。

指向“日用品补库”：

> 对厨房纸这类低风险、高频商品，它根据实际消耗周期，在月度授权内自动比较、凑单和补货，不需要每次打扰用户。

指向“稀缺品抢购”：

> 对演唱会门票这类稀缺资源，它可以在可信库存出现时自动锁位和支付。人的反应速度是秒级，Agent 的执行可以是毫秒级。

指向“高值蹲低价”：

> 对相机这类高值商品，它会验证是不是全年真实低价、是不是先涨后降，但超过自动购买阈值后仍然需要用户确认。

指向“盯二手商品”：

> 对二手商品，它可以公开匿名需求，让卖家 Agent 主动提交方案，再通过成色归一、自动砍价和专业验机降低风险。

### 本段收束句

> 关键不是让 Agent 什么都替用户决定，而是把不同风险的消费，放进不同的授权边界。

### 听众此时应该理解

主动服务不是失控的自动购买，而是按风险、金额和场景分级授权。

---

## 3:15–4:00 Inbox：商业信息先由 Agent 审查，而不是直接争夺人的注意力

### 页面操作

1. 点击“Inbox”；
2. 选择 DJI 低价消息；
3. 指向“Agent 评价”和“建议存入记忆”；
4. 再选择被拦截的服饰营销消息。

### 讲述

> 商家仍然可以主动营销，但它不能直接购买用户的注意力。
>
> 所有商业信息先进入消费者 Agent 的 Inbox，由消费者自己设定接收条件。

指向 DJI 消息：

> 这条相机信息通过了三个检查：价格确实处于全年低位、不是先涨后降、渠道和官方保修都可验证。所以 Agent 判断它是有价值的商业信息。

指向“建议存入记忆”：

> Agent 只把“官方渠道最低可信价 4699 元”这个事实写入长期记忆，不会把“限时、爆款”这样的营销措辞写进去。

切换到被拦截消息：

> 这条消息没有具体商品参数，也没有库存证据，商家信用又低于用户设定的阈值，因此在到达人之前就被拦截。

### 本段收束句

> 在这里，营销权属于消费者，商家必须提供有价值、可验证的信息，才能进入人的视野。

### 听众此时应该理解

AgentShop 没有消灭营销，而是把营销从“购买注意力”变成“证明信息价值”。

---

## 4:00–5:15 商家端：从购买流量变成争夺决策准入

### 页面操作

1. 点击顶部“商家端”；
2. 保持在“交易战情”；
3. 指向左侧“历史买家记录”；
4. 指向中间“买家 Agent 会话”；
5. 指向右侧“胜负核心依据”和“编队内部影响”。

### 讲述

> 现在我们切到同一笔交易的商家视角。
>
> 商家看到的不是访客数、点击率和广告曝光，而是正在发生的买家 Agent 需求，以及自己是否有资格进入它们的决策集。

指向左侧记录：

> 每一个买家 Agent 的需求、提案、协商和履约结果都会完整留档，商家可以回放任何一次竞争。

指向中间会话：

> 这里能看到买家公开了哪些约束、销售 Agent 提交了什么提案、双方如何议价，以及最后做出了什么履约承诺。

指向右侧胜负依据：

> 这次胜出不是因为买了更多广告，而是因为履约信用更高、所有硬约束都满足，并且在议价后仍然保持合理毛利。

指向编队内部影响：

> 背后也不是一个万能 Agent，而是一支员工 Agent 编队：销售 Agent 负责提案，供应链 Agent 锁定库存，客服 Agent 承诺响应，财务 Agent 管理结算和毛利。
>
> 如果最后发生延误，责任可以归因到具体环节，而不是笼统地说“服务不好”。

### 本段收束句

> 对商家来说，新的核心经营指标不是曝光，而是进入高质量买家决策集的概率。

### 听众此时应该理解

AgentShop 同时为商家提供获客、销售协作、履约管理和复盘能力。

---

## 5:15–6:25 四种 A2A 销售机制：商家如何获得客户

### 页面操作

1. 点击“销售机制”；
2. 在“约束锚定式精准推销”中点击“运行机制演示”；
3. 指向右侧排名、成交概率、获客成本变化；
4. 依次点击另外三种机制，只讲一句核心差异。

### 讲述

> 商家在这个网络里有四种获得客户的方式。

运行“约束锚定式精准推销”：

> 第一种是精准提案。销售 Agent 监听匿名需求、装配链上信用证据，再生成满足约束的方案。这个案例里，决策集排名从第四升到第一，成交概率从 24% 提升到 72%，获客成本从 86 元下降到 12 元。

点击“合约裂变式分销”：

> 第二种是链上返佣分销。商家发布公开、自动结算的返佣合约，中继 Agent 会主动寻找匹配需求。佣金只在真实成交后产生，并且即时结算。

点击“履约声誉排序竞争”：

> 第三种是声誉排序。商家通过提升交付和售后获得更高自然准入，而不是购买排名。履约越好，未来获客越容易。

点击“广播推销”：

> 第四种是受限广播。消息必须先通过消费者的 Inbox 条件，再由买家 Agent 独立判断价值。刚才消费者端看到的相机低价信息，就是同一个案例。

### 本段收束句

> 这四种机制共同完成一个变化：商家不再为“被看见”付费，而是为“值得被选择”持续优化。

### 听众此时应该理解

这是新的市场分配机制，不只是一个商家聊天机器人。

---

## 6:25–7:20 网络拓扑：为什么必须有 RepChain

### 页面操作

1. 点击顶部“网络拓扑”；
2. 保持“剧情演示”；
3. 依次点击第 1、2、3、4、5、6 步；
4. 在第 4 步和第 5 步适当停留。

### 讲述

> 最后把这笔交易放回整个网络里看。

第 1–3 步：

> 买家 Agent 发布匿名需求，多个门店 Agent 提案，网络根据买家偏好和商家信用决定谁能进入决策集。

第 4 步：

> 当门店库存不足时，它的供应链 Agent 会变成一个买家，继续向仓储和工厂 Agent 采购。
>
> 所以这不是消费者和门店之间的一层网络，而是一套可以递归到整个供应链的 A2A 协作协议。

第 5 步：

> 工厂、仓储、物流和门店的每一段履约都会形成鉴证。如果上游延误，它会沿供应链影响下游商家的信用，不能把责任隐藏在黑箱里。

第 6 步：

> 一次交易的真实结果最终回到 RepChain，更新商家信用和市场准入。后来所有买家 Agent 都可以复用这次经验，不需要重复踩坑。

### 本段收束句

> RepChain 的核心价值，是把一个人的一次试错，变成整个网络可以复用的信用资产。

### 听众此时应该理解

信用链不是为了增加一个区块链概念，而是为了给 Agent 的自动决策提供可信输入。

---

## 7:20–7:50 结尾：用一句商业判断收束

### 页面操作

停留在网络拓扑，不再切换页面。

### 讲述

> 我们不是在做另一个购物助手。
>
> 我们在验证一种新的商业基础设施：消费者把决策权交给忠于自己的 Agent，商家用真实履约争夺决策准入，每笔交易又继续增强整个网络的信用。
>
> 当商业主体逐渐从人变成 Agent，稀缺资源将不再是人的注意力，而是进入买家决策集的资格。
>
> AgentShop 希望成为这套 Agent-to-Agent 商业网络的交易与信任层。

> AgentShop 是连接买家 Agent、商家 Agent 和供应链 Agent 的交易网络；RepChain 是它们共同信任的履约底座。

# 常见追问与建议回答

## 1. 这和普通购物 AI 有什么区别？

> 普通购物 AI 主要帮助用户搜索和推荐，交易仍然发生在原有平台逻辑里。AgentShop 代理的是完整交易，并且把商家准入、议价、履约和信用回写放进同一个 A2A 网络。

## 2. 如何保证消费者 Agent 不被商家收买？

> 消费者 Agent 的商业利益必须结构性归属于买家，不能依靠推荐佣金排序。商家提供的营销信息只作为待验证输入，真正进入决策权重的是 RepChain 上的客观履约数据。

## 3. 商家能不能刷信用？

> 信用不是人工评价，而是交易结束后由订单、物流、仓储、质检和售后节点共同生成。不同买家的鉴证权重还会根据自身信用调整，大量低信用小号无法获得与高信用买家或专业鉴证 Agent 相同的影响力。

## 4. 为什么一定需要区块链？

> 原型验证的是“跨主体共享、不可由单方修改的履约账本”这一产品需求。最终使用公链、联盟链还是可信账本，需要根据性能、成本和治理继续验证，我们不会在原型阶段假装技术选型已经完成。

## 5. 新商家没有历史信用怎么办？

> 这是冷启动问题。可行方向包括保证金、专业鉴证、担保方信用和阶梯式放量。新商家可以进入市场，但在积累真实履约之前不能直接获得与成熟商家相同的准入权重。

## 6. 你们怎么赚钱？

> 当前原型优先验证交易机制和双边价值。商业模式可以来自商家 Agent 基础设施订阅、履约鉴证服务或结算服务，但不能依赖向商家出售消费者推荐排序，否则会破坏买家 Agent 的利益一致性。

## 7. 最先从哪个市场切入？

> 最适合的起点是参数相对标准、履约结果可量化、购买频率较高的品类，例如日用品补库、标准化 3C 或企业采购。它们更容易验证自动比较、授权代买和客观鉴证闭环。

## 8. 当前原型哪些部分还不是真实系统？

> 当前对话、交易和经营指标采用可重复的合成数据，用于稳定演示完整机制。真实库存、支付、跨主体结算和信用链技术选型仍需要后续接入和验证。

# 演示表达注意事项

## 建议反复使用的表达

- “不是推荐，而是完整交易代理。”
- “不是购买曝光，而是争夺决策准入。”
- “不是主观五星评价，而是客观履约鉴证。”
- “把一个人的一次试错，变成全网共享的信用资产。”
- “Agent 的授权边界由消费者决定。”

## 尽量避免的表达

- 不要一开场连续解释 C-Agent、S-Agent、DRN、RepChain 四个缩写；
- 不要把所有页面指标逐个念一遍；
- 不要宣称真实区块链、真实支付或真实市场预测已经完成；
- 不要说 Agent 展示了“完整思维链”，应称为“可审计决策日志”；
- 不要在消费者故事尚未建立前，先讲复杂的供应链或信用算法；
- 不要把“去中心化”作为唯一价值，先讲可信履约和利益一致性。

## 现场节奏

- 一次只讲一个观点，讲完再点击；
- 数字只念有对比意义的变化；
- 页面动画运行时停顿，让听众先看再听解释；
- 每个模块最后必须有一句收束，不要直接跳到下一页；
- 如果操作出现延迟，继续讲业务逻辑，不要向听众解释技术故障。
